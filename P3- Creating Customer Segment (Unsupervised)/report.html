Implementation: Outlier Detection¶

Detecting outliers in the data is extremely important in the data preprocessing step of any analysis. The presence of outliers can often skew results which take into consideration these data points. There are many "rules of thumb" for what constitutes an outlier in a dataset. Here, we will use Tukey's Method for identfying outliers: An outlier step is calculated as 1.5 times the interquartile range (IQR). A data point with a feature that is beyond an outlier step outside of the IQR for that feature is considered abnormal.

In the code block below, you will need to implement the following:

  • Assign the value of the 25th percentile for the given feature to Q1. Use np.percentile for this.
  • Assign the value of the 75th percentile for the given feature to Q3. Again, use np.percentile.
  • Assign the calculation of an outlier step for the given feature to step.
  • Optionally remove data points from the dataset by adding indices to the outliers list.

NOTE: If you choose to remove any outliers, ensure that the sample data does not contain any of these points!

Question 6¶

What are the advantages to using a K-Means clustering algorithm? What are the advantages to using a Gaussian Mixture Model clustering algorithm? Given your observations about the wholesale customer data so far, which of the two algorithms will you use and why?

Answer: KMeans will return hard boundaries. In each step, the output is that an element belongs or doesn't belong to a cluster. What changes between the steps is to which cluster the observation has been assigned.

A Gaussian Mixture Model, on the other hand, uses a expectation-maximization algorithm to work with probabilities. It's return is a distribution of probabilities for each observation regarding which cluster they might be part of.

From the analysis we have done so far, the dataset doesn't seem to have clearly distinguishable clusters. This could make the Gaussian Mixture Model a good option.

However, we need also to consider the business requirement, which is to "equip the distributor with insight into how to best structure their delivery service to meet the needs of each customer". The results of the kMeans are easier to interpret by non-technical individuals and easier to work with. So I would choose kMeans over GMM due to the straightforward results, with a hard cut classification.

Question 10¶

Companies will often run A/B tests when making small changes to their products or services to determine whether making that change will affect its customers positively or negatively. The wholesale distributor is considering changing its delivery service from currently 5 days a week to 3 days a week. However, the distributor will only make this change in delivery service for customers that react positively. How can the wholesale distributor use the customer segments to determine which customers, if any, would react positively to the change in delivery service?
Hint: Can we assume the change affects all customers equally? How can we determine which group of customers it affects the most?

Answer:

The whole idea of A/B testing is running an experiment. To run an experiment, you need a control group. A is the control group, B is the group in which you make the changes, or vice versa.

To ensure you get statistically significant results, there are three elements to be defined in the experiment:

  • the independent variable, the variable that will be modified.
  • dependent variable, what is measured
  • controlled variables.

So lets say you define an experiment to measure to the impact of changing the delivery time (independent variable) on customer satisfaction (dependent variable). A will be the control group, B the experiment group.

In simpler terms, you show your customers two versions of the product, a 5 days/week and a 3 days/week delivery time, and compare the response to each of the products.

There are two ways here which you could use cluster information:

1. You can use the cluster information to control for the customer type, ensuring that the control and the test group have the same proportion of customers from segment 0 and from segment 1.

Control for variable is the attempt to reduce the effect of confounding variables on an observational study ([wiki])(https://en.wikipedia.org/wiki/Controlling_for_a_variable).

2. This specific case states the distributor will only make this change in delivery service for customers that react positively. That means the customer is considering the option of applying the change only to one group. So the best option is to segregate customers into two segments, run two separate A/B tests, one for each segment, and evaluate the results for each group independently.

If, for any of the segments, the customer satisfaction is not affected or is affected positively by the change in delivery time, that means the change can be implemented for that segment.
